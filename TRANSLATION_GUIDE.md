# FTC ドキュメント 日本語翻訳ガイドライン (TRANSLATION_GUIDE)

このガイドラインは、公式 FTC ドキュメント（ftcdocs）の日本語翻訳の品質と一貫性を保つために作成されました。貢献される方は、必ずこのルールに従って翻訳作業を行ってください。

---

## 1. 翻訳の基本方針

### 1.1 文体（トーン＆マナー）

* **ですます調:** ドキュメント全体を通じて、**「です・ます」調**に統一します。
* **敬称:** 読者への呼びかけは「皆さん」や「読者の方」とし、「あなた」や「君」といった呼びかけは避けてください。
* **表現:** 技術文書として、正確で簡潔な表現を心がけます。冗長な言い回しや感情的な表現は使用しないでください。

### 1.2 固有名詞と著作権

* **著作権の維持:** 原文にある著作権表示、ライセンス、免責事項はそのまま残し、その翻訳（もしあれば）を併記してください。
* **FIRST/FTCの表記:** 「FIRST」および「FIRST Tech Challenge (FTC)」の表記は、原則として**英語のまま**残します。

---

## 2. 用語の取り扱いルール

用語は「和訳しないFTC・API固有の語」と「和訳またはカタカナで統一する一般的な技術用語」に分類し、一貫性を保ちます。

### 2.1 和訳しない用語（FTC・API固有の語）

**原則として、コード内のクラス名、API名、および FTC 固有の概念は和訳せず、英語のまま残します。**

* **理由:** 実際のコードや、英語の公式フォーラムで検索する際に、和訳するとかえって混乱が生じるためです。
* **表記:** 必ず**太字**にして英語のまま残します。

| カテゴリ | 用語（英語のまま残す） | 備考 |
| :--- | :--- | :--- |
| **API/クラス名** | **`OpMode`**, **`LinearOpMode`**, **`Telemetry`**, **`HardwareMap`**, **`Robot Controller`** | コードに登場する固有名詞です。 |
| **制御モード** | **`Autonomous`**, **`TeleOp`** | 競技フィールドでの操作モードです。 |
| **ハードウェア** | **`REV Robotics Control Hub`**, **`REV Robotics Expansion Hub`**, **`DRIVER STATION`**, **`IMU`** | 製品名やセンサーの名称です。 |

> **【重要】和訳しない用語に関する注記**
> 上記リストに記載がない場合でも、**特定のソフトウェア名、ハードウェア名、あるいはコード（API）に関連する固有名詞**や、**和訳することでコードとの整合性が失われる**と判断されるワードについては、翻訳せず**英語のまま（太字にして）**残してください。
> 新しい用語を発見した場合は、PRを出す前に**用語統一リストに追加**することを推奨します。

### 2.2 和訳またはカタカナ表記する用語（一般的な技術語）

一般的な技術用語や、すでに日本語コミュニティで定着しているカタカナ語は、**以下のように統一して使用します**。

| 英語 | 統一訳語/カタカナ語 | 備考 |
| :--- | :--- | :--- |
| **Debugging** | **デバッグ** | 「デバッギング」ではなく「デバッグ」と表記します。 |
| **Feature** | **機能** | 「フィーチャー」ではなく「機能」と和訳します。 |
| **Variable** | **変数** | 「ヴァリアブル」ではなく「変数」と和訳します。 |
| **Parameter** | **パラメーター** | 「パラメータ」のように長音符号を省略せず「パラメーター」と表記します。 |
| **Framework** | **フレームワーク** | カタカナ語として使用します。 |
| **Build** | **ビルド** | ソフトウェアの構築作業として使用します。 |
| **Component** | **コンポーネント** | 部品、構成要素を意味するカタカナ語です。 |
| **Repository** | **リポジトリ** | 「レポジトリ」ではなく「リポジトリ」と表記します。 |

## 3. 作業ルールと貢献方法

### 3.1 ブランチ運用

* **作業ブランチ:** すべての翻訳作業は、**`docs-ja`** ブランチで行ってください。`main` ブランチへの直接プッシュは禁止です。
* **本家との同期:** 本家ドキュメントが更新された場合は、まず `main` ブランチに変更を取り込み、その後 `main` から `docs-ja` へマージ（またはリベース）して作業を続けてください。

### 3.2 コミットとプルリクエスト (PR)

* **コミットメッセージ:** コミットメッセージは**英語**で簡潔に記述します（例: `Translate chapter 3 introduction`）。
* **PRの作成:** 翻訳が完了したファイルや章があれば、`docs-ja` からプルリクエストを作成し、レビューを依頼してください。PRのタイトルと概要には、**日本語で**変更点を明確に記述します。
* **排他制御:** 同時に同じファイルを翻訳するのを避けるため、作業に着手する際は該当する Issue を立てるか、チャットなどで宣言してください。
