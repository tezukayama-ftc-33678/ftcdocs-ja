# FTC Docs 日本語翻訳ガイド

## 1. 基本方針

### 1.1 文体
- **です・ます調**を使用してください
- 丁寧で分かりやすい表現を心がけてください

### 1.2 技術用語の扱い
- API名、クラス名、メソッド名は**英語のまま**保持し、**太字**で表記してください
  - 例: **OpMode**, **DcMotor**, **Telemetry**
- 一般的な技術用語は日本語またはカタカナで翻訳してください
  - 例: configuration → 設定、sensor → センサー、motor → モーター

## 2. フォーマット

### 2.1 コードとAPI
- コードブロック内のコードは翻訳しないでください
- コメントは翻訳してください
- API名は**太字**で表記してください

### 2.2 画像とリンク
- 画像のパスやリンクは変更しないでください
- 画像のキャプションは翻訳してください

## 3. 用語の統一

- **GLOSSARY.md**を参照して、用語の一貫性を保ってください
- 新しい技術用語が出てきた場合は、GLOSSARY.mdに追加してください

## 4. 品質チェック

- 翻訳後は必ず読み返して、自然な日本語になっているか確認してください
- 原文の意味が正確に伝わっているか確認してください
