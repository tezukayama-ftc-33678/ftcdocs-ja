# ライセンスとロゴ使用に関するガイド

このガイドでは、日本語翻訳版FTCドキュメントにおけるライセンスとFIRSTロゴの使用について説明します。

## よくある質問（FAQ）

### Q1: このドキュメントを公開して法的に問題ありませんか？

**A: はい、問題ありません。**

理由：
1. **原文のライセンス**: 原文は BSD 3-Clause License の下で公開されています
2. **再配布の許可**: このライセンスは、ソースコードとバイナリ形式の再配布を明示的に許可しています
3. **翻訳の位置づけ**: 翻訳は「派生物」として扱われ、同じライセンス条件で配布できます

### Q2: FIRSTのロゴはそのまま使っても大丈夫ですか？

**A: はい、ドキュメント内での使用は問題ありません。**

理由：
1. **ドキュメンテーション内での使用**: 原文のドキュメントに含まれるロゴは、ドキュメントの一部として使用が許可されています
2. **商標の適正使用**: ロゴは削除せず、原文通りに使用します
3. **誤解を避ける表示**: 非公式翻訳であることを明示することで、FIRSTの公式承認を誤認させることを防ぎます

### Q3: 記事内に権利表記はどうすれば良いですか？

**A: 以下の場所に権利表記を配置しています。**

#### 1. トップページ（index.rst）
```rst
.. note::
   **📋 ライセンスと著作権について**
   
   * 原文: © 2022 FIRST Tech Challenge (BSD 3-Clause License)
   * 翻訳: Team 33678 Tezukayama (同ライセンス)
   * FIRST®、FIRST® Tech Challenge、およびロゴは FIRST® の登録商標です
   * 詳細は LICENSE-JA.md をご覧ください
```

#### 2. フッター（全ページ）
- 原文の著作権表示: "© 2025 FIRST"
- 翻訳者の表示: "日本語翻訳: Team 33678 Tezukayama"
- 免責事項ボックス（日本語版のみ表示）

#### 3. LICENSE-JA.md
詳細なライセンス情報と免責事項を記載

#### 4. README.md
プロジェクトの概要と位置づけを明記

### Q4: 商標表記は必要ですか？

**A: はい、適切に表記しています。**

表記方法：
- `FIRST®` - 登録商標であることを示す®マーク付き
- `FIRST® Tech Challenge` - 同様に®マーク付き
- LICENSE-JA.mdに「FIRSTの登録商標である」ことを明記

### Q5: Read the Docsで公開するために追加で必要な手続きはありますか？

**A: 特別な許可申請は不要です。**

理由：
1. **オープンライセンス**: BSD 3-Clause Licenseは自由な再配布を許可しています
2. **非公式であることの明示**: 非公式翻訳であることを明確に表示しています
3. **原文へのリンク**: 公式ドキュメントへのリンクを保持しています

ただし、以下を推奨します：
- FIRSTコミュニティに翻訳版の存在を通知（任意）
- フィードバックを受け付ける窓口を明記

## BSD 3-Clause Licenseの要件

このライセンスで守るべき3つの条件：

### 1. 著作権表示の保持
✅ **対応済み**
- 原文の著作権表示を保持: "Copyright (c) 2022, FIRST Tech Challenge"
- LICENSEファイルをそのまま保持

### 2. 免責事項の表示
✅ **対応済み**
- LICENSEファイルに原文の免責事項を保持
- LICENSE-JA.mdに追加の免責事項を記載

### 3. 無断でのエンドースメントの禁止
✅ **対応済み**
- "非公式翻訳"であることを明示
- "FIRSTによって公式に承認されていない"ことを明記

## FIRSTロゴの使用ガイドライン

### 使用可能な場面

✅ **OK: ドキュメント内での使用**
- ヘッダーロゴ: `FIRSTTech_iconHorz_RGB_reverse.png`
- ファビコン: `FIRSTicon_RGB_withTM.ico`
- PDF表紙: `Latex_Logo_FTC.png`
- その他、原文に含まれるロゴ

✅ **OK: 原文通りの配置**
- ロゴの位置やサイズを変更しない
- ロゴの色やデザインを変更しない

### 使用に注意が必要な場面

⚠️ **注意: 新しいロゴの追加**
- 原文にないロゴの追加は避ける
- 翻訳プロジェクト専用のロゴは作成しない

⚠️ **注意: ロゴと翻訳の関係**
- ロゴの近くに「非公式翻訳」の表記を入れる（フッターで対応）

### 使用してはいけない場面

❌ **NG: 商用利用**
- ロゴを使った商品販売
- ロゴを使った有料サービス

❌ **NG: 誤解を招く使用**
- 公式プロジェクトであるかのような表示
- FIRSTの推奨を受けているかのような表示

## 実際のファイル配置

### 必須ファイル

```
ftcdocs-ja/
├── LICENSE                    # 原文のBSD 3-Clause License（必須）
├── LICENSE-JA.md              # 翻訳版の補足説明（必須）
├── README.md                  # 非公式であることの明記（必須）
├── PUBLISHING.md              # 公開手順（推奨）
└── docs/
    └── source/
        ├── index.rst          # トップページの警告と注記（必須）
        ├── _templates/
        │   └── footer.html    # フッターの免責事項（必須）
        └── assets/
            └── FIRST*.png/ico # FIRSTロゴ（原文から維持）
```

### フッターの実装

`docs/source/_templates/footer.html` に以下を追加済み：

```html
{%- if language == 'ja' %}
<div role="contentinfo" style="background-color: #fff3cd; ...">
  <strong>⚠️ 非公式翻訳版の注意事項:</strong>
  この日本語翻訳は <strong>非公式</strong> であり、FIRST® の公式ドキュメントではありません。
  ...
</div>
{%- endif %}
```

## 推奨事項

### 1. 定期的な原文の確認
- 原文のライセンスやTerms of Serviceに変更がないか確認
- 変更があった場合は翻訳版も更新

### 2. コミュニティとの対話
- GitHubのIssuesでフィードバックを受け付け
- 誤訳や改善提案を積極的に取り入れる

### 3. 透明性の維持
- 翻訳プロセスを公開（README等で説明）
- 変更履歴をGitで管理

### 4. 公式との連携（任意）
- FIRSTコミュニティフォーラムで翻訳版を紹介
- 公式チームからのフィードバックがあれば対応

## トラブルシューティング

### ケース1: ロゴ使用について問い合わせがあった場合

回答例：
> 「このロゴは原文のFIRST Tech Challenge公式ドキュメントに含まれているものです。
> 私たちはそのドキュメント全体を翻訳しており、ロゴもドキュメントの一部として
> 原文通りに使用しています。非公式翻訳であることは明確に表示しています。」

### ケース2: 公式からの削除要請があった場合

対応方針：
1. すぐに要請に従う準備をする
2. 対話を通じて状況を説明
3. 必要に応じて、ロゴの削除や表記の変更を実施

### ケース3: ライセンス違反の指摘があった場合

対応手順：
1. 具体的な指摘内容を確認
2. BSD 3-Clause Licenseの要件を再確認
3. 不足があれば即座に修正
4. 修正内容を記録・公開
