# 翻訳問題トラッカー使用ガイド

## 概要

FTC Docs 日本語版の翻訳問題を記録・管理するためのツールです。
ブラウザで記事を読みながら、おかしい箇所を簡単に記録できます。

## 使い方

### 1. ツールを開く

ローカルで HTML ファイルを直接開くか、ビルドしたドキュメントの `_static/issue_tracker.html` にアクセスしてください。

```bash
# ローカルで直接開く場合
open docs/source/_static/issue_tracker.html

# または、ブラウザで以下のパスを開く
file:///path/to/ftcdocs-ja/docs/source/_static/issue_tracker.html
```

### 2. 問題を記録する

フォームに以下の情報を入力してください：

- **ページタイトル** (必須): 例: `index.rst (トップページ)`、`rookie_teams.rst`
- **ページURL** (任意): ビルドしたドキュメントのURLを記録できます
- **問題の種類** (必須): 以下から選択
  - 翻訳の誤り
  - 構造の崩れ
  - ビルドエラー
  - その他
- **問題の詳細** (必須): どのような問題があるか具体的に記述してください

入力後、「📝 問題を記録」ボタンをクリックします。

### 3. 記録された問題を確認する

- 記録された問題は画面下部に一覧表示されます
- 統計情報（総問題数、種類別の問題数）も表示されます
- 各問題の右上にある「削除」ボタンで個別に削除できます

### 4. エクスポート機能

問題リストをエクスポートして、他のメンバーと共有したり、GitHub Issueを作成したりできます：

- **💾 JSONエクスポート**: JSON形式でダウンロード（プログラムで処理する場合に便利）
- **📄 Markdownエクスポート**: Markdown形式でダウンロード（GitHub Issueや文書として使いやすい）

### 5. データ保存について

- すべてのデータはブラウザの LocalStorage に保存されます
- ブラウザを閉じても、データは保持されます
- ただし、ブラウザのキャッシュをクリアするとデータが失われるため、定期的にエクスポートすることをお勧めします

## ワークフロー例

1. ビルドしたドキュメントをブラウザで開く
2. 別のタブで問題トラッカーを開く
3. ドキュメントを読みながら、問題を見つけたら問題トラッカーに記録
4. 一通り確認が終わったら、Markdown形式でエクスポート
5. エクスポートしたMarkdownファイルを GitHub の Issue として投稿するか、修正作業に使用

## 技術詳細

- **フレームワーク**: バニラ JavaScript (依存関係なし)
- **ストレージ**: LocalStorage API
- **スタイル**: カスタムCSS（レスポンシブデザイン対応）
- **ブラウザ互換性**: モダンブラウザ (Chrome, Firefox, Safari, Edge) に対応

## トラブルシューティング

### データが保存されない

- ブラウザのプライベートモード（シークレットモード）では LocalStorage が無効化される場合があります
- 通常モードで開き直してください

### エクスポートがうまくいかない

- ポップアップブロッカーが有効になっている可能性があります
- ブラウザの設定で、このサイトのポップアップを許可してください
